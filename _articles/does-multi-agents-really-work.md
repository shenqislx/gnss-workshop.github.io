从去年开始，关于 multi-agents/agent-team 的讨论就没断过，到 clawdbot（现在已经更名为 OpenClaw）问世，更是将这个话题推到一个高潮。很多人都在讨论，**多个 agent 协同工作，是不是比单个 agent 更强大？** 



# 多个 agent 协同工作，真的更强大吗？

我个人的感受是，对于普通个人开发者，如果忽视实际需求，盲目追求多个 agent 协同工作，反而可能会适得其反，导致效率降低，甚至完全失控。
原因有多个方面，既有使用者的因素，也有工具本身的因素。
一方面，每个 agent 都有自己的知识面和注意力范围，如果没有明确的规则去协调它们的工作，它们很容易就会跑偏，甚至互相干扰，导致整个工作流变得混乱不堪。
